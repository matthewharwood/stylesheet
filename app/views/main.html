Dependency Injection

DI is a software design pattern that allows the passing of a dependency / service to an object. This allows for a loosely coupled system where this service (or dependencies) is defined independently. This means that the class using the dependency has no knowledge of the internals of the service being called.

Decoupling and the separation of the behaviour from the data.
